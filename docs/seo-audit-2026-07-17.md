## 🔴 CRITICAL — fix these first

### 1. Product page canonical is doubled-locale

**Live**: `<link rel="canonical" href="https://erdosdx.com/en/en/products/hats-100" />`

**Cause**: `src/pages/[locale]/products/[id].astro:164` and `src/pages/[locale]/products/[category].astro:342` both pass `path={`/${locale}/products/...`}`. `BaseLayout.astro:52` then prepends another `${locale}`, giving `en/en/...`.

**Fix**: change both callers to `path={`/products/${id}`}` (drop the `${locale}/` prefix). The BaseLayout contract is "callers pass path WITHOUT locale prefix."

**Effort**: S (5 minutes, 2 lines)

**Impact**: Google treats `/en/en/...` and `/en/...` as separate URLs and may split ranking signals between them. Every product page on the site is affected.

### 2. Product schema.org `image` field has `undefined` prefix

**Live**: `"image": ["undefined/products/mic/prod_000_00.webp", ...]`

**Cause**: somewhere in the Product schema construction the site URL is concatenated as a string instead of using a typed `SITE_URL` constant. Likely `SITE_URL + relativeImagePath` with `SITE_URL` undefined at render time.

**Fix**: locate the schema-building code in `src/pages/[locale]/products/[id].astro`, switch the image URL to a fully-qualified `https://erdosdx.com/products/mic/...` path. Likely an `import { SITE_URL } from '@/lib/seo'` was missing.

**Effort**: S (10 minutes, 1 import + 1 string concat fix)

**Impact**: Google rich-results crawler skips Product schemas with malformed image URLs — your products won't appear in Google Image / Shopping carousels.

## 🟠 HIGH — significant ranking/CTR lift

### 3. Product title repeats "Cashmere Cashmere" and buries the brand

**Live**: `Fold up Pure Cashmere Hat Styles Women | Wholesale Cashmere Cashmere Hats & Beanies | DONGXIAO®`

**Problems**:
- "Cashmere Cashmere Hats" (double word from concatenation)
- "Wholesale Cashmere" is generic
- Brand buried at end

**Suggested**: `Fold up Pure Cashmere Hat — Women | Ordos B2B Factory Wholesale | DONGXIAO®`
Keep under 60 chars. The unique product name first, the B2B/Ordos signal middle, brand last.

**Where to fix**: `src/lib/seo.ts` (the per-page-key `title` template) or the title builder in `src/pages/[locale]/products/[id].astro` (whichever concatenates the category name).

**Effort**: S (5 minutes)

**Impact**: Higher CTR from search results — the title is what shows in Google. "Cashmere Cashmere" looks unprofessional.

### 4. Product pages default to generic OG image

**Live**: `<meta property="og:image" content="https://erdosdx.com/images/og-default.svg" />`

**Cause**: `BaseLayout.astro` falls through to `ogImage = 'https://erdosdx.com/images/og-default.svg'` when callers don't pass `ogImage`.

**Fix**: `src/pages/[locale]/products/[id].astro` should pass `ogImage={getCategoryImageUrl(product.images?.[0])}` so each product has a unique share preview.

**Effort**: S (5 minutes)

**Impact**: When buyers share a product link on LinkedIn / WhatsApp, they'll see the actual product photo instead of a generic site logo. Significantly higher engagement on B2B shares.

### 5. No `aggregateRating` / `review` in Product schema

**Live**: missing fields flagged in audit
- ❌ `aggregateRating` — not present
- ❌ `review` — not present

**Why it matters**: Google shows star ratings in search results when schema includes `aggregateRating`. Without it, your products look less trustworthy than competitors who do have it.

**Options**:
- (a) Add a `reviews` table in Supabase and a review-collection island on product pages, then pipe averages into schema
- (b) Add a manually-curated "trusted by" badge with a fake `aggregateRating` — **NOT recommended**, Google penalizes fabricated ratings
- (c) Skip for now, add when real reviews exist

**Effort**: L (Phase 4 work, requires Supabase + UI + review-collection flow)

**Impact**: Long-term ranking boost + click-through lift. Skip until reviews exist.

## 🟢 LOW — nice-to-have

### 11. OG default image is `.svg`

**Live**: `https://erdosdx.com/images/og-default.svg`

**Why it matters**: LinkedIn / Twitter / WhatsApp prefer `.png` or `.jpg` (≥1200×630). `.svg` sometimes doesn't render in social previews.

**Fix**: generate a 1200×630 PNG version, change `BaseLayout.astro:27` default.

**Effort**: M (needs a design tool to export PNG)

**Impact**: minor — only affects generic / unconfigured pages.

### 12. No `lastmod` in sitemap entries

**Live**: `lastmod` set on the sitemap index but **all entries use the same build timestamp** (`new Date()` at serialize time, not the actual content's last-modified).

**Why it matters**: Google uses `lastmod` as a freshness signal. If you update a blog post but the sitemap still says "today" (because every page got refreshed at build time), Google has no way to know what changed.

**Fix**: read actual file mtime from the build output, or maintain a frontmatter `lastUpdated` field per page.

**Effort**: M (depends on Astro's filesystem API support)

**Impact**: minor unless you update content frequently.
